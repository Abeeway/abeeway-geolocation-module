◆ CLI_COMMAND_FUNC_REGISTER

#define CLI_COMMAND_FUNC_REGISTER (   cmd_s,
  help_s,
  function_p,
  access_u8 
)
Value:
__attribute__((section(".commands."#cmd_s),used)) \
static const cli_parser_cmd_t __cmdfun_##cmd_s = { \
.command = #cmd_s, .help = (help_s), \
{ .func = (function_p) }, \
.action = cli_parser_action_execute, \
.access = (access_u8) }

Register a main command against the CLI manager.

Parameters
cmd_sCommand name (string)
help_sCommand help (string)
function_pCallback called when the command has been parsed.
access_u8Access level required to execute this command.
Note
The CLI_COMMAND_REGISTER() macro requires a specific linker script modification within the SECTIONS block. Sorting by name is recommended, as otherwise the commands appear in the order the linker finds them - that is, randomly.

The linker command should have the following lines: 

 .commands : {
     . = ALIGN(4);
     _cli_command_table = .;
    KEEP(*(SORT_BY_NAME(.commands.*)));
     LONG (0)
 } > FLASH
