aos_lpm_requester_callback_t

Callback called before entering the low power mode.

Parameters
argUser argument
Returns
true if the requestor can sleep (move to STOP1/2) mode, false otherwise
Note
Some action may be required by the requester before entering a low power mode. The requester has to do them under this callback.

◆ aos_lpm_event_t

enum aos_lpm_event_t

Low power manager event notification type.

These events are meant to allow various drivers to turn off or on whatever can be to reduce power consumption. They are sent immediately before and after the WFI instruction, so no system services are available - only register accesses.

Enumerator
aos_lpm_event_test_sleep 

Ask a client if we may go to sleep.

aos_lpm_event_pre_sleep 

The system is about to go to sleep.

aos_lpm_event_post_sleep 

The system has just woken up.

(10.2mA) No low power mode. The chip is in high speed mode.

aos_lpm_mode_sleep 

( 5.1mA) Sleep mode

aos_lpm_mode_stop1 

( 9.2uA) Medium low power. The chip can move in STOP1.

aos_lpm_mode_stop2 

( 5.0uA) Lowest power mode. The chip can move in STOP2

Number of low power modes.
