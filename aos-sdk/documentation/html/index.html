This manual is addressed to developers wishing to write an application on the top of the SDK.

Introduction

General architecture

The SDK is composed by several third party components such as the STM32 HAL, STM32 BLE, freeRTOS, Semtech LoRaWan protocol stack and LR1110 driver.
The SDK also contains several modules, drivers and facilities provided by Abeeway.
The term AOS, which can be found in the documents and the code, stands for Abeeway Operating System. It contains hardware component drivers and offers APIS and services for applications.
Note that AOS is based around freeRTOS and is built as a layered architecture that can be depicted as follow.

Hardware Abstraction Layer

The Hardware abstraction layer or HAL manages the hardware. It offers basic functionalities.
Basics facilities such as FIFO management or basic GPIO configuration reside at this level.
The HAL is split into two components:

  • ST HAL: HAL provided by the manufacturer (ST MicroElectronic)
  • AOS HAL: Abeeyway Operating System

These lower layers are accessible by other parts of the operating system.
Usually the application layer does not need to access directly the HAL. It usually uses services or enhanced services.
In some cases however, it could be useful for the application to access the HAL. That's the reason why the HAL is not closed regarding the application layer.
